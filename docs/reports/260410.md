## 1. Summary

The robomimic guidance adapter now uses chunk-derived sliding windows inside
`DiffusionPolicy.grad_log_prob(...)` instead of reducing each chunk position to
an isolated repeated-state / repeated-action query.

The public contract is unchanged:

- `state` is still `[B, H, Dobs_feat]`
- `action` is still `[B, H, Da]`
- the returned score is still `[B, H, Da]`

## 2. Windowed Surrogate

For observation conditioning, the adapter constructs a causal window of length
`To = observation_horizon` for each chunk step `i`:

- window slot `j` uses chunk index `i + j - (To - 1)`
- indices are clamped into `[0, H - 1]`
- the current chunk state therefore lands in the final observation slot

For action conditioning, the adapter constructs a robomimic-aligned action
window of length `Tp = prediction_horizon` for each chunk step `i`:

- window slot `j` uses chunk index `i + j - action_start_index`
- `action_start_index = observation_horizon - 1`
- indices are clamped into `[0, H - 1]`
- the queried chunk action therefore lands at the executable robomimic action
  slot selected by rollout

## 3. Remaining Limits

This is still an approximation to robomimic's true chunk score:

- the denoiser is still queried at the fixed
  `DiffusionPolicyScoreConfig.score_timestep`
- the adapter still does not receive the sampler's true prefix history outside
  the current chunk, so left-boundary conditioning uses in-chunk edge
  replication rather than exact prefix states
