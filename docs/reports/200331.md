## 3. Metrics

Training objective:

$$\begin{align}
\mathcal{L}_{\text{DDPM}}
=
\mathbb{E}_{x_0, \epsilon, t}
\left[
\left\|
\epsilon - \epsilon_\theta(x_t, t, c)
\right\|_2^2
\right]
\end{align}$$

Evaluation metric:

$$\begin{align}
\text{RMSE}
&=
\sqrt{
\frac{1}{BWD}
\sum_{b=1}^{B}
\sum_{w=1}^{W}
\sum_{d=1}^{D}
\left(\hat{y}_{bwd} - y_{bwd}\right)^2
}
\end{align}$$

Reported slices:

- full transition RMSE
- state RMSE
- action RMSE
- `robot0_eef_pos` RMSE

## 6. Interpretation

Likely issue:

- not diffusion optimization alone
- not dataset construction alone
- not simple low-dimensional prediction alone
- specifically: conditioning injection for 19D robomimic state

Current failure hypothesis:

- prefix-state conditioning is too weak
- in-painting formulation is mismatched to short low-dimensional history
- observed context and future prediction target are not cleanly separated


## 7. Next Step

Proposed direction:

- replace prefix in-painting with explicit FiLM-style conditioning
- feed observed state history as external context
- keep future prediction target unchanged

Conceptual denoiser:

$$\begin{align}
\epsilon_\theta(x_t, t, c_t)
\end{align}$$

where $c_t$ is the encoded observed-state context.

First implementation:

- flatten or lightly encode the observed prefix
- inject context into the denoiser explicitly
- test on the same 19D robomimic setup

De-prioritized for now:

- quaternion-specific conditioning tricks
- richer state geometry engineering

Reason:

- if explicit conditioning works, current diagnosis is validated
- longer-term goal is image-embedding conditioning, not hand-tuned low-dimensional state encoding
