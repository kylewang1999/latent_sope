## 1. Summary

This note records the current guidance interpretation check relevant to the
local FiLM-conditioned chunk diffuser and the possibility of using chunk
`predict-x0` instead of chunk `predict-epsilon`.

Short version:

- upstream SOPE always forms an $x_0$-space estimate before constructing the
  reverse-step Gaussian
- policy guidance is an action-only score term, padded with zeros in the state
  coordinates
- the perturbation is applied to the reverse-step mean returned by
  `p_mean_variance(...)`, which itself is computed from the current $x_0$
  estimate
- this structure still makes sense under FiLM conditioning and chunk
  `predict-x0`; what changes is only how $\hat\tau_\theta^0$ is produced

## 2. Upstream Guidance Path

In upstream SOPE, `predict_start_from_noise(...)` always produces an $x_0$-space
estimate:

$$\begin{align}
\hat\tau_\theta^0(\tau^k, k, c)
\approx
\mathbb{E}\!\left[\tau^0 \mid \tau^k, c\right].
\end{align}$$

If the chunk model uses `predict-epsilon`, this is obtained by converting the
predicted noise into $\hat\tau_\theta^0$. If the chunk model uses
`predict-x0`, the model output is already interpreted as $\hat\tau_\theta^0$.

The reverse-step Gaussian is then formed from $(\tau^k, \hat\tau_\theta^0)$:

$$\begin{align}
p_\theta(\tau^{k-1} \mid \tau^k, c)
=
q\!\left(\tau^{k-1} \mid \tau^k, \hat\tau_\theta^0(\tau^k, k, c)\right).
\end{align}$$

Equivalently, `p_mean_variance(...)` returns the reverse-step mean and variance
after this conversion step. So the guidance path is:

1. produce $\hat\tau_\theta^0$
2. convert it into the reverse Gaussian moments
3. perturb the reverse-step mean
4. sample using the original reverse variance

## 3. Policy Guidance Term

For diffusion-policy guidance, the action score is interpreted in the usual
`predict-epsilon` form:

$$\begin{align}
\nabla_{a_t^k} \log \pi_k\left(a_t^k \mid s_t\right)
\approx
-\frac{\hat{\epsilon}_\theta\left(a_t^k, k, s_t\right)}{\sqrt{1-\bar{\alpha}_k}}=:\hat g_{a,t}^\pi.
\end{align}$$

Upstream SOPE then pads this with zeros in the state coordinates to match the
chunk shape:

$$\begin{align}
g_{\mathrm{act}}(\tau^k)
=
\big(
[0_{d_s}, \hat g^\pi_{a,1}],
\ldots,
[0_{d_s}, \hat g^\pi_{a,H}]
\big).
\end{align}$$

Showing only the positive-guidance case for illustration, the actual reverse
update is therefore of the form

$$\begin{align}
\mu_\theta^k(\tau^k, c)
\longrightarrow
\mu_\theta^k(\tau^k, c) + \eta_k\, g_{\mathrm{act}}\!\left(\mu_\theta^k(\tau^k, c)\right),
\end{align}$$

where $\mu_\theta^k$ is the reverse-step mean returned by
`p_mean_variance(...)`. In code, the guidance is added directly to this reverse
mean, not directly to $\hat\tau_\theta^0$.

## 4. Relevance To Local FiLM And Chunk `predict-x0`

This upstream reading still transfers to the local FiLM-conditioned chunk
diffuser:

- FiLM conditioning changes how context enters the denoiser, but it does not
  change the DDPM logic that maps the denoiser output to $\hat\tau_\theta^0$
  and then to a reverse-step mean
- switching the chunk model from `predict-epsilon` to `predict-x0` changes only
  how $\hat\tau_\theta^0$ is obtained
- the action-only policy guidance term can still be added on top of the
  reverse-step mean exactly as before

So the current working interpretation is:

$$\begin{align}
\text{FiLM-conditioned chunk reverse step}
\quad + \quad
\text{action-only policy guidance}
\end{align}$$

with the same guidance structure under chunk `predict-x0`, provided the policy
score term is still interpreted consistently.
