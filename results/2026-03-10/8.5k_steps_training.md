## Diagnosis: Why 500 epochs isn't enough

**The fundamental problem is data throughput, not epochs.** With only 17 batches per epoch:
- 500 epochs = 8,500 steps (each seeing 64 samples)
- SOPE does 5,000 steps per epoch WITH REPLACEMENT from a much larger dataset
- We're seeing each training chunk ~500 times (8500 steps × 64 / 1250 chunks ≈ 435 presentations), but only 17 unique batches per pass
- The model sees the same 17 batches repeatedly, but with batch_size=64 and shuffle=True, each epoch presents different groupings

**However**, the loss IS still decreasing, meaning the model hasn't memorized the data. This suggests the model capacity / architecture may also be a factor:
- Our dim_mults=(1, 2) → channels [32, 64] — very small
- SOPE reference uses dim_mults=(1, 4, 8) → channels [32, 128, 256] — much larger
- Our model: 252,698 parameters. SOPE reference likely 10-50x larger.

## Concrete Next Steps (priority order)

### 1. Increase model capacity (HIGH PRIORITY)
The model is too small to learn a 26-dimensional (19 state + 7 action) temporal distribution.
- Change `dim_mults` from `(1, 2)` to `(1, 4, 8)` to match SOPE reference
- This will increase parameters from ~250k to ~2-5M
- May need to reduce batch size if GPU memory is tight

### 2. Add with-replacement sampling (HIGH PRIORITY)
Instead of 17 batches/epoch (1 pass), sample 500+ batches/epoch with replacement.
- Modify training loop: instead of `for batch in dataloader`, sample N random batches
- Target: 5000 steps/epoch × 100 epochs = 500,000 steps (matching SOPE)
- With 17 batches per epoch currently, would need ~29,000 epochs to match SOPE — clearly we need with-replacement

### 3. Both together (RECOMMENDED)
- Larger model (dim_mults=(1, 4, 8)) + with-replacement sampling (5000 steps/epoch, 100 epochs)
- This matches the SOPE reference setup
- Expected training time: ~500k steps × (time/step) — need to benchmark, but likely 30-60 min on P100

### 4. Data diversity concern for 500k run
50 rollouts ≈ 1,300 chunks. With 500k steps × batch_size 64, each chunk is seen ~24,600 times. SOPE uses D4RL datasets with ~125k chunks (100x more diversity). Risk: model memorizes chunks perfectly (low chunk L2) but fails at stitching due to poor generalization to unseen conditioning states. **If chunk L2 is great but OPE is still bad → collect 200 rollouts (4x more diversity).**

### 5. Lower priority
- Reducing diffusion steps (64 instead of 256) — requires retraining but may help given the model is weak
- Post-hoc quaternion normalization — won't help since ALL dimensions blow up, not just quaternions
- 6D rotation representation — premature optimization; fix training budget first
